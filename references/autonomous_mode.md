# H5 Forge Reference - 全自动执行策略

`h5f-a` 是全自动执行策略，不是独立任务模式。

## 目标

减少用户反复确认。缺少但不阻塞的信息，由 controller 根据项目规则、相似实现和 H5/Web 主流实践选择推荐方案，继续推进直到完成。

## auto_assumption

自动补齐的决定必须显式记录：

```text
- auto_assumption：未提供状态方案，检测到项目主流是 Zustand，按 feature-local store 推进
```

来源优先级：

1. 当前项目规则卡
2. 相似实现
3. 当前项目主流代码模式
4. H5/Web 常规工程实践

## 必须中断确认

- 安全、隐私、支付、鉴权生产策略
- 删除数据或不可逆迁移
- 全项目架构切换
- write scope 不清晰
- 会影响线上行为但无法本地验证

## 阶段门禁

全自动不是无门禁。它只是把普通确认替换为推荐方案自动确认；高风险确认仍由用户决定。
