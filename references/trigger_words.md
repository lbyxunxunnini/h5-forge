# H5 Forge Reference - Trigger Words (Single Source)

这个文件是 `h5-forge` 触发词的**唯一权威列表**。其他文档（`SKILL.md`、`README.md`、`QUICKSTART.md`、`CHEATSHEET.md`、`skill_visibility.md` 等）必须引用本文件，不允许复述完整列表，避免多源维护。

## 正式触发词（按写法分组）

h5-forge 只接受以下 5 种正式写法。识别要求：**触发词必须出现在用户消息的首个 token 位置**（行首，token 边界由空格或标点界定）。

### 1. `h5f-` 标准流程

匹配规则：`^h5f-(?:\s|$)`（行首 `h5f-` 后必须是空白或字符串结束）

适用：新页面、新模块、PRD、设计图、重构、迁移等需要走完整结构化流程的任务。

```text
h5f- 帮我做一个订单列表页
h5f- 这是一个迭代中的 H5/Web 项目，先扫描结构
```

### 2. `h5f-fast` 快速路径

匹配规则：`^h5f-fast(?:\s|$)`

适用：小改动、局部 bug、轻量/中等任务，发现风险再升级。

```text
h5f-fast 改一下登录页按钮颜色
```

### 3. `h5f-a` 全自动路径（紧凑写法）

匹配规则：`^h5f-a(?:\s|$)`

适用：缺少非阻塞信息时由 controller 采用推荐方案继续推进。

```text
h5f-a 新建商品详情页，缺的自动补完
```

### 4. `h5f a` 全自动路径（带空格写法，等价于 `h5f-a`）

匹配规则：`^h5f\s+a(?:\s|$)`

存在原因：减少对手感的依赖；语义与 `h5f-a` 完全一致。

```text
h5f a 新建商品详情页
```

### 5. `/h5-forge` 斜杠命令

匹配规则：`^/h5-forge(?:\s|$)`

适用：宿主提供 slash 命令面板时；行为等价于 `h5f-`。

```text
/h5-forge 帮我 review 这个页面
```

## 非触发词（明确不识别）

以下写法**不再识别为触发词**，输入时按普通对话处理：

- `h5f -`（h5f 加空格加破折号）—— 视觉上等价于 `h5f-`，无独立必要
- `h5f `（h5f 加空格，无后续 a）—— 黑名单式排除不安全（容易撞 `h5f next page` 等正常输入）
- `h5-forge`（不带斜杠）—— 改用 `/h5-forge` 或 `h5f-`
- `使用 h5-forge` / `按 h5-forge 工作模式处理` —— 改为显式触发词

历史变更说明：此前 description 中包含 `"h5f -"`、`"h5f "`、`"h5-forge"`、`"使用 h5-forge"`、`"按 h5-forge 工作模式处理"` 等写法，自规则收敛起不再支持，但用户若使用上述自然语言表达"按结构化流程处理"的意图，可走 `SKILL.md` 中"用户明确要求按结构化流程处理"的进入条件。

## 防误触发硬规则

- 锚定行首：触发词必须在用户消息**首个 token 位置**，不识别消息中段出现的 `h5f-`
- token 边界：触发词后必须是空白字符或字符串结束（防止匹配 `h5f-cake`、`h5f-orange` 等）
- 用户在引用代码块、URL 中出现的 `h5f-` 不触发

## 文档引用规则

- `SKILL.md` frontmatter `description` 字段：必须内联正式触发词，因为 skill 召回只读取 frontmatter，不会为了触发去读取本文件；详细匹配规则仍以本文件为准
- `SKILL.md` 正文："命中路由 - 进入条件" 节引用本文件，正文里只列匹配规则要点
- `README.md` / `QUICKSTART.md` / `CHEATSHEET.md`：用例中可使用具体触发词，但不维护"完整列表"，需要时链接到本文件
- 所有其他 reference 文件：需要列触发词时引用本文件路径，不重复

## 维护规则

- 新增/废弃触发词必须先改本文件，再同步到引用方
- description 收敛到 ≤ 220 字符（宿主截断风险线），且必须保留正式触发词入口
- 不在多个文档同时维护触发词清单
