# H5 Forge Reference - New Project Profile Selection

当 H5 Forge 进入一个新项目时，不应只做“通用规则起步”，而应显式判断是否需要从已有项目记忆或模板画像中选择初始规则来源。

## 目标

在新项目里，为用户提供三种初始规则来源：

1. 复制某个已有项目规则卡作为初始模板
2. 使用当前项目扫描结果作为起点
3. 使用通用 H5/Web 规则 + 全局偏好

## 推荐流程

### 步骤 1：读取全局偏好

先读取：

- `memory/global_preferences.yaml`

用于恢复跨项目稳定偏好。

### 步骤 2：扫描当前项目

优先扫描：

- 目录结构
- 已有依赖
- 已有路由方式
- 已有状态管理痕迹
- 已有主题 / l10n / 网络层结构

### 步骤 3：匹配候选来源

从两个来源找候选：

1. `memory/projects/*.rule_card.yaml`
2. `memory/profiles/*.yaml`

判断哪些候选与当前项目最接近，并给出简短说明。

### 步骤 4：向用户给出选择

推荐输出格式：

```text
我已经完成新项目的初步扫描，当前可选的初始规则来源有：

1. 复制已有项目规则卡
   - project_a：feature 目录 + Pinia/Zustand
   - project_b：module 目录 + Redux/Pinia

2. 使用当前项目扫描结果
   - 依据当前已有依赖、目录和路由结构建立规则卡

3. 使用通用 H5/Web 规则 + 全局偏好
   - 不复用旧项目规则，只保留你的稳定个人偏好

请确认优先使用哪一种。
```

### 步骤 5：生成项目级规则卡

无论用户选哪一种，最终都要生成该项目自己的：

- `memory/projects/<project_name>.rule_card.yaml`

它才是后续任务真正使用的项目记忆。

## 关键约束

- 可以复制已有规则作为初始模板
- 不能把旧项目规则直接当成新项目真相
- 当前项目扫描结果必须作为校正依据
- 如果当前项目与模板冲突，以当前项目现状为准
