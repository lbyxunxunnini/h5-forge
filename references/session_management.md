# H5 Forge Reference - 会话管理

这个文件定义 `.h5-forge/session.md` 的格式、写入时机、读取时机和恢复逻辑。

## Session 文件路径

项目根目录下 `.h5-forge/session.md`。

## 格式

### 基础格式

适用于单任务或无子任务场景：

```markdown
# H5 Forge Session

- 规则卡：已加载 / 未加载
- 规则卡摘要：2-3 个关键字段（如状态管理、路由、网络层）
- 当前任务：轻量任务 / 中等任务 / 大任务
- 任务模式：页面开发 / 代码审查 / 迁移辅助 / i18n/a11y / 迭代中项目扫描 / 新项目初始化 / 需求理解
- 任务阶段：Phase N（描述） / 无阶段
- 阶段进度：Phase 1 ✓ / Phase 2 ✓ / Phase 3 进行中 / 无阶段
- 最近操作：具体做了什么
- 更新时间：YYYY-MM-DD HH:mm
```

### 带子任务清单格式

当任务拆分为多个子任务时（如一个 sprint 包含多个页面改造），**必须**使用清单格式跟踪每个子任务：

```markdown
# H5 Forge Session

- 规则卡：已加载 / 未加载
- 规则卡摘要：Redux + React Router + axios
- 当前任务：大任务
- 任务模式：页面开发
- 任务阶段：Sprint A — Day 2
- 更新时间：2025-01-15 14:30

## 子任务清单

- [x] D1-1: login.html (01a 改造)
- [x] D1-2: splash.html (01b 改造)
- [ ] D2-1: phone_login.html (02 改造)
- [ ] D2-2: profile.html (03 改造)
- [ ] D3: settings.html (20 补齐)
```

规则：

- 每个子任务用 `- [x]` 或 `- [ ]` 标记完成状态
- 子任务 ID 保持和执行计划一致
- 括号内标注设计稿编号或任务类型
- 已完成的子任务放前面，未完成的放后面（可选，但推荐按 ID 顺序）

## 写入时机

### 必须写入的时刻

1. **模式判定完成时** — 写入任务分级、任务模式、规则卡状态
2. **阶段切换时** — 写入当前阶段和进度（Phase N 完成 → Phase N+1 开始）
3. **每个子任务完成时** — 更新对应子任务的 `[ ]` → `[x]`，同步更新"最近操作"和"更新时间"
4. **整个任务完成时** — 写入完成状态

### 不需要写入的时刻

- 每读一个文件
- 每调一次工具
- 中间思考过程
- 与用户确认细节

### 关键原则

**"不需要每改一个文件都写"指的是实现过程中的中间步骤，不是指子任务级别的完成标记。** 子任务完成是一个明确的里程碑，必须立即更新。

判断标准：如果用户问"这个做完了吗"，你的回答是"做完了"而不是"还在做"，那就该写入了。

## 读取时机

### 必须读取的时刻

1. **对话被压缩后恢复** — 先读 session.md，恢复工作模式
2. **暂停恢复** — 用户回来继续时
3. **新消息触发 h5-forge 时** — 检查是否有未完成任务

### 读取后的恢复逻辑

```
读取 session.md
  → 文件不存在？ → 视为新任务，走正常任务路由
  → 存在且有未完成子任务？ → 恢复到记录的模式和阶段
  → 存在但所有子任务已完成？ → 视为新任务
  → 存在但"最近操作"为空或很久以前？ → 先询问用户是否继续
```

### 弱继续表述处理

用户说"继续之前的工作"、"接着做"、"然后呢"等弱继续表述时：

1. 先读 `.h5-forge/session.md`
2. 如果有未完成子任务 → 轻量恢复，输出一行当前模式，列出未完成子任务
3. 如果没有未完成任务 → 视为新任务，走正常任务路由
4. **不要假设用户还记得上下文** — 恢复时简要说明上次做到哪了

### 恢复输出格式

恢复时输出：

```
[h5-forge] 模式：轻量任务，恢复上次进度

已完成：D1-1, D1-2
待完成：D2-1, D2-2, D3

下一个要做什么？
```

不要输出大段解释，简洁列出状态即可。

## 防误判规则

1. **不要把"最近操作"当成完成证明** — 只有子任务清单中 `[x]` 才算完成
2. **不要跨 session 假设** — 每次恢复都要重新读取 session.md
3. **不要忽略清单格式** — 如果 session.md 有 `## 子任务清单` 段落，必须以清单状态为准，不能只看"最近操作"
4. **新任务覆盖旧任务** — 用户开始新任务时，清空或覆盖旧 session，不要残留旧子任务

## 维护规则

- session.md 是临时状态文件，不是文档
- 不要把 session.md 提交到 git（应加入 .gitignore）
- 项目初始化时创建空 session.md
- 任务全部完成后可以保留（供下次恢复参考），也可以清空
