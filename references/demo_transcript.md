# H5 Forge Demo Transcript

这个文件提供可复制的演示对话，帮助外部用户快速理解 `h5-forge` 的输出形态。它是 demo transcript，不是验证日志；真实试跑记录仍以 [validation_log.md](validation_log.md) 为准。

## Demo 1：迭代中项目首次接入

```text
用户：
h5f- 这是一个迭代中的 H5 项目。先扫描项目结构，输出规则卡草案，不要先写代码。

助手：
[h5-forge] 模式：迭代中项目扫描
- 升级原因：首次接入迭代中 H5/Web 项目，需要先扫描结构、状态管理、路由、网络层和组件边界，生成规则卡草案后再执行开发任务。
[h5-forge] 阶段：S1 需求确认
[h5-forge] 需求分析师：当前目标是建立项目规则理解，不进入实现。
[h5-forge] 阶段：S2 方案确认
[h5-forge] 架构设计师：将扫描 package.json、src/ 前两层、路由配置、网络层入口、状态文件和已有规则文件。
[h5-forge] 架构设计师：检测到 React、Vite、Zustand、React Router、Axios 和 Vitest 信号；规则卡草案将按 evidence 标注 confidence。
[h5-forge] 本轮完成：已完成项目扫描摘要和规则卡草案；未修改业务代码；下一步等待用户确认规则卡草案。
```

```text
[h5-forge] 需求分析师：已确认核心范围：商品基础信息、轮播图、规格选择、底部购买入口。仍需确认当前最高优先级问题：规格选择是否需要影响价格和库存展示？
```

```text
助手：
[h5-forge] 模式：架构级任务

```

## Demo 4：全自动页面开发

```text
用户：
h5f-a 新建一个商品详情页，包含轮播图、价格、规格选择和底部购买按钮，缺少的部分按推荐方案自动做完。

助手：
[h5-forge] 全自动：已启用 h5f-a，非阻塞缺口将采用推荐方案推进；安全、不可逆或高风险架构决策才中断确认。
[h5-forge] 模式：页面开发
- 升级原因：新增页面，涉及页面结构、规格选择状态、底部购买入口和路由接入。
[h5-forge] 阶段：S1 需求确认
[h5-forge] 需求分析师：规格选择影响价格和库存的规则未指定，自动采用推荐方案：选择规格后同步刷新价格、库存和购买按钮可用状态。
[h5-forge] 阶段：S2 方案确认
[h5-forge] UI 设计师：未提供设计图，自动采用推荐方案：沿用项目现有详情页卡片风格，首屏为轮播图 + 价格信息 + 规格选择 + 底部固定购买栏。
[h5-forge] 架构设计师：未指定状态管理，自动沿用规则卡主流方案 Zustand，并按现有路由入口注册详情页。
[h5-forge] 阶段：S4 实现中
[h5-forge] 页面工程师：按自动冻结方案实现页面、状态接入和路由注册。
[h5-forge] 阶段：S5 验证中
[h5-forge] 验证工程师：执行 lint / 单元测试或项目等价验证。
[h5-forge] 全自动摘要：本轮自动采用 3 项推荐方案：规格联动价格库存；详情页卡片风格；Zustand 页面级状态接入。
```

## Demo 使用原则

- Demo 可以用于 README、发布说明和 issue 引导。
- Demo 不得计入真实验证案例。
- 如果 demo 和实际协议冲突，以 `SKILL.md` 与 `references/task_runtime_prompt.md` 为准，并同步修订 demo。
