# H5 Forge Reference - 可见性与会话状态

连续对话中，用户需要知道 h5-forge 是否在工作。采用任务级标记，不干扰但可感知。

## 角色切换信号

### 任务开始时

模式标志表见 SKILL.md "工作模式标志"。用一行短提示说明当前模式。

完整流程必须在模式标志后补一行升级原因，避免把中等任务误拉进重流程：

```text
[h5-forge] 模式：页面开发
- 升级原因：新增页面 + 状态管理接入 + 路由注册
```

如果这是对上一轮误路由的纠正，先补一行短说明，再继续正确流程：

```text
[h5-forge] 页面工程师：上一轮路由判断有误，现已纠正。
[h5-forge] 模式：页面开发
- 升级原因：...
```

### 大任务切换阶段时

只在阶段变化时给一行提示：

- `[h5-forge] 当前阶段：UI 解析`
- `[h5-forge] 当前阶段：实现设计`
- `[h5-forge] 当前阶段：页面开发`

### 角色输出标注

大任务中，每个角色的关键决策和建议必须带角色标签：

```
[h5-forge] 需求分析师：提取 8 个约束，核心是订单列表+筛选+分页
[h5-forge] UI 设计师：拆为 3 个组件，列表用 virtual list / List rendering
[h5-forge] 架构设计师：订单状态用 Pinia/Zustand，全局用 Redux/Pinia
[h5-forge] 页面工程师：开始生成代码，预计 3 个文件
```

规则：

- 格式：`[h5-forge] 角色名：内容`
- 每个角色独占一段，不混在同一段里
- 架构设计师的决策行必须标注（决策权归属依据）
- 页面工程师的实施建议和验证清单必须标注
- 轻量任务强制标注 `[h5-forge] 页面工程师：`，中等任务同理

### 初始化分流时

如果当前进入的是项目初始化而不是直接任务执行：

- `[h5-forge] 检测到迭代中项目且无规则卡，建议先初始化项目规则`
- `[h5-forge] 检测到新项目，建议先选择初始规则来源`

### 不要做的事

- 不要每一步都长篇解释
- 不要默认输出完整角色交接文档
- 不要在没有具体升级原因时进入完整流程
- 不要在用户已经指出误路由后，再反问是否进入正确流程
- 不要为了展示 skill 存在感而制造无效仪式感
- 不要自行省略 `[h5-forge]` 标记（这是必选输出，不是可选装饰）
- 不要每读一个文件或每调一次工具就插一句 `[h5-forge]`，轻量任务只在开始和结束时各输出一次

### 上游返回的可见性

上游返回机制触发时，只有架构师有架构影响才显示架构师对话，其他返回静默流转：

- 页面工程师 → UI 设计师（复杂 UI 缺失）：显示 UI 设计师的索要输出
- 页面工程师 → 需求分析师（复杂功能缺失）：显示需求分析师的确认输出
- 页面工程师 → 需求分析师（用户要求回到需求阶段）：优先显示需求分析师输出，不再继续实现态
- 架构师评估无影响：不显示架构师对话，上游补充后直接下发
- 架构师评估有影响：显示架构师调整方案

## 默认输出格式

输出格式遵循 SKILL.md "任务分级"定义（轻量任务 / 中等任务 / 大任务各有不同输出要求）。

## 可见性标记

### 标记规则

- 运维直通：回复开头带一行 `[h5-forge] 页面工程师：运维直通，快速执行`
- 轻量任务：回复开头带一行 `[h5-forge] 页面工程师：轻量任务，直接执行`
- 中等任务：回复开头带 `[h5-forge] 中任务 + 基于什么规则（如有规则卡）`
- 大任务：每个角色切换时带 `[h5-forge] 角色名：正在做什么`，结束时带 `[h5-forge] 完成`
- 后续对话中 h5-forge 继续介入时，每条关键回复开头带 `[h5-forge]` 标记

### 标记格式

```
[h5-forge] 页面工程师：运维直通，快速执行
[h5-forge] 页面工程师：轻量任务，直接执行
[h5-forge] 页面工程师：基于规则卡（读取 rule_card.yaml 中的 state_management + routing 等关键字段）生成代码
[h5-forge] 需求分析师：正在拆解 PRD
[h5-forge] 完成，已生成订单列表页骨架
```

如果用户说“这是一个需求”或“需要需求分析”，下一条可见输出必须改为：

```text
[h5-forge] 需求分析师：上一轮已偏到执行态，现回到需求阶段
```

规则卡摘要只取 2-3 个关键字段（如状态管理、路由、网络层），不要把整个规则卡内容塞进标记里。没有规则卡时写 `[h5-forge] 基于项目现有风格生成代码`。

### 不标记的情况

- 用户问的是纯知识问题（"MobX 怎么用"），h5-forge 未介入项目代码
- 闲聊、确认、追问等不涉及任务执行的对话

## 会话状态

维护 `.h5-forge/session.md`，记录当前会话的 h5-forge 工作状态。用户可以随时问"h5-forge 现在是什么状态"来查看。

会话格式、写入时机、读取时机和恢复逻辑详见 [session_management.md](session_management.md)。

关键规则：

- 子任务清单中 `[ ]` → `[x]` 的更新是**每个子任务完成时的必选动作**，不是"关键节点才更新"
- 恢复时以子任务清单状态为准，不以"最近操作"为准
- 不主动输出，不每轮回复都展示
