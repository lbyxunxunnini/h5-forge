# H5 Forge Reference - 启动握手

启动握手是 h5-forge 命中后的默认输出行为，定义什么时候输出、输出什么。

## 什么时候输出完整握手

只有在以下情况，才对用户显式输出完整启动握手：

1. 当前项目未初始化
2. 当前项目没有正式规则卡
3. 当前项目首次被 H5 Forge 接管
4. 当前项目规则卡刚生成、刚迁移或刚失效
5. 前端协作 skills 状态未就绪、未映射或未确认

## 完全态

当前项目已经有正式规则卡 → 视为**已被 H5 Forge 接管**。

只有当以下两项同时成立时，才视为进入**完全态**：

1. 当前项目存在正式规则卡
2. 前端协作 skills 状态已就绪（当前会话已明确可见 前端协作 skills，或已选定本地协作技能目录且检测到可协作 前端协作 skills）

被接管后的默认行为：

- 仍然在内部执行项目状态检查、规则卡检查和 前端协作 skills 探测
- 进入完全态后，不再重复向用户输出完整握手日志
- 默认直接进入下一环节
- 只有在规则冲突、规则卡失效、技能缺失或用户明确要求查看握手细节时，才重新展开握手输出

## 完整握手输出格式

第一行输出：

- `[h5-forge] 模式：启动握手`

然后至少补三条最小结果日志：

1. 项目类型
   - `- 项目类型：迭代中项目`
   - `- 项目类型：新项目`
2. 规则卡状态
   - `- 规则卡：已加载`
   - `- 规则卡路径：.h5-forge/projects/<project>.rule_card.yaml`
   - `- 项目状态：已初始化`
   - `- 规则卡：未发现，准备初始化`
   - `- 规则卡草案路径：.h5-forge/projects/<project>.rule_card_draft.yaml`
   - `- 项目状态：未初始化`
   - `- 当前判断来源：项目扫描 / 当前代码结构 / 会话上下文`
3. 前端协作 skills 协作能力
   - `- 前端协作 skills：当前会话可见`
   - `- 前端协作 skills：已检测到本地协作技能目录`
   - `- 前端协作 skills 状态：已就绪`
   - `- 前端协作 skills：未检测到，将使用内置流程`
   - `- 前端协作 skills 状态：未就绪`

如果需要，再补最后一条当前分流结论：

- `- 当前模式：迭代中项目扫描`
- `- 当前模式：新项目初始化`
- `- 当前模式：直接进入当前任务`
- `- 当前模式：需求理解 -> 设计包`

规则卡路径只能在当前目标项目根目录内精确解析：

1. `.claude/.h5-forge/projects/<project>.rule_card.yaml`
2. `.trae/.h5-forge/projects/<project>.rule_card.yaml`
3. `.agent/.h5-forge/projects/<project>.rule_card.yaml`
4. `.h5-forge/projects/<project>.rule_card.yaml`

禁止把 `~/.claude/projects/.../memory/*.yaml`、其他项目目录中的规则卡、当前项目目录下其他项目名的规则卡当作已加载规则卡。

## 进入工作阶段日志

完成启动握手或静默检查后，开始真正处理任务时，输出：

- `[h5-forge] 模式：进入工作阶段`

随后标明当前输入模型：

- `- 输入模型：只给 PRD`
- `- 输入模型：只给设计图`
- `- 输入模型：PRD + 设计图`
- `- 输入模型：上下文不足`
- `- 输入模型：直接开发任务`

常见场景：

1. **迭代中项目 + 无规则卡** → 先输出完整启动握手，再进入输入模型日志
2. **迭代中项目 + 有规则卡 + 前端协作 skills 未就绪** → 继续输出最小握手日志，再进入输入模型日志
3. **已进入完全态** → 不再重复完整握手，直接输出 `[h5-forge] 模式：进入工作阶段` + 输入模型
4. **新项目初始化后** → 先完成初始化，再输出进入工作阶段日志
