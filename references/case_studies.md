# H5 Forge Reference - Case Studies

这个文件定义 H5 Forge 推荐维护的真实案例类型，用于后续验证 skill 的有效性。

## Case 1: 迭代中项目首次接入

目标：

- 扫描迭代中 H5/Web 项目
- 识别目录结构、命名、状态管理、组件边界、路由和 API 模式
- 输出项目规则摘要与规则卡

验证点：

- 摘要是否接近真实项目
- 规则卡是否能支撑后续页面开发
- 是否错误继承了其他项目规则

## Case 2: 迭代中项目新页面开发

目标：

- 在已有项目里新增一个真实业务页面
- 先做需求理解、UI 解析、实现设计，再写代码

验证点：

- 页面结构树是否合理
- 文件结构与命名是否符合项目主流
- 复用建议是否正确
- 是否需要补测试

## Case 3: 新项目初始化

目标：

- 扫描一个新 H5/Web 项目
- 在“已有项目规则卡 / 当前扫描结果 / 通用规则 + 全局偏好”之间选择初始来源
- 生成该项目自己的规则卡

验证点：

- 是否正确区分模板与真实项目规则
- 是否避免错误复制旧项目规范

## 建议记录的指标

- 结构树命中率
- 命名 / 目录归属准确率
- 复用建议采纳率
- 用户纠错次数
- 测试建议命中率
