## Persistent Principles

无论任务大小，都保持以下底线：

1. 用户最终拍板
2. 业务目标优先
3. 架构一致性与可维护性优先于表面 UI 还原
4. 团队规则优先于个人偏好
5. 低置信度推断不能强行当成团队规则
