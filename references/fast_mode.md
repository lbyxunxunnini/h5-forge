# H5 Forge Reference - 快速执行策略

`h5f-fast` 是执行策略，不是独立任务模式。

## 目标

减少冷启动和沟通成本，让小任务和中等任务更快完成。

## 执行规则

- 先运行或模拟 `scripts/project_snapshot.py` 获取冷启动摘要。
- 轻量任务最多读 1-3 个关键文件。
- 轻量任务最多输出开始和完成 2 条主要日志。
- 中等任务采用“15 分钟内完成策略”：扫描相似实现 → 直接改 → 最小验证。
- 发现需求缺口、UI 结构决策、架构边界风险时必须升级。

## 禁止

- 不能因为快速入口跳过安全确认。
- 不能跳过验证。
- 不能把新页面、大功能、架构迁移强行压成轻量任务。

## 输出示例

```text
[h5-forge] 模式：轻量任务 / 快速
[h5-forge] 页面工程师：已完成，验证通过
```
