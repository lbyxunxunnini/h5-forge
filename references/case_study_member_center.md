# H5 Forge Case Study - Member Center Page

## 场景

项目类型：

- 迭代中 H5/Web 业务项目

输入：

- 会员中心页面 PRD
- 会员中心页面设计图
- 现有项目代码库

目标：

- 新增会员中心页面
- 尽量复用现有页面模式
- 保证命名、目录、状态管理和路由方式符合项目主流

## 推荐流程

### 1. 需求理解

产出：

- 页面目标
- 核心用户路径
- 必须覆盖的状态
- 待确认项

### 2. UI 解析

产出：

- 页面结构树
- 区块边界
- 组件边界草案
- 空态 / loading / error 覆盖判断

### 3. 实现设计

产出：

- 模块归属方案
- 文件结构方案
- 命名方案
- 状态管理接法
- 复用策略
- 是否需要路由、l10n、测试委托

### 4. 页面开发

产出：

- 页面骨架
- 状态接入骨架
- 接口占位
- 关键交互入口

### 5. 质量检查

至少检查：

- 是否需要 `npm run lint`
- 是否需要 Component 测试
- 是否存在明显反模式

## 这个案例的价值

- 适合验证迭代中项目接入能力
- 适合验证“需求 -> UI -> 结构 -> 代码”链路是否打通
- 适合验证复用策略是否真正有帮助
