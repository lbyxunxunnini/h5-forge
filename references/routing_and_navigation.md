# H5 Forge Reference - Routing And Navigation Rules

这个文件定义项目级路由与导航规范，补足“技术实现之外的项目约束”。

## 关注范围

H5 Forge 在项目内应明确：

1. 路由命名规范
2. 页面参数传递方式
3. 深度链接处理方式
4. 嵌套路由边界
5. 页面内导航动作归属

## 路由命名

默认建议：

- 路由名与页面语义保持一致
- 路由路径风格在项目内保持统一
- 不在同一项目里混用多套命名风格，除非明确是历史迁移

命名风格本身以项目主流为准，H5 Forge 不强推 kebab-case 或 camelCase。

## 参数传递

建议明确：

- 简单标量参数如何传
- 复杂对象是否允许直接传整对象
- 需要恢复或分享的页面是否要求序列化参数

默认建议：

- 可路由恢复的页面优先传稳定参数，而不是直接传整对象引用

## 深度链接

如果项目支持深度链接，应明确：

- 由哪一层解析
- 未登录或权限不足时如何回退
- 参数不完整时如何处理

## 嵌套路由

需要明确：

- 哪些页面允许自己管理子导航
- 哪些页面必须服从全局路由层
- Tab 页与子栈的边界在哪里

## 页面内导航动作

默认建议：

- 页面动作触发导航时，尽量通过项目统一路由层或封装调用
- 不要在页面深处散落大量硬编码跳转逻辑
