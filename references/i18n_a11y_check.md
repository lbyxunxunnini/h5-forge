# H5 Forge Reference - 国际化与无障碍检查

在代码生成或审查时，检查国际化（i18n）和无障碍（a11y）支持。

## 国际化检查

### 检查时机

- 代码生成后：检查是否有硬编码中文/英文字符串
- 代码审查时：检查字符串是否已提取到 l10n 文件
- 规则卡有 i18n 要求时：自动启用

### 检查项

1. **硬编码字符串**：页面中是否有直接写死的中文/英文文本
2. **日期格式**：是否有硬编码的日期格式（应使用 intl 包）
3. **数字格式**：是否有硬编码的数字格式（应使用 NumberFormat）
4. **图片资源**：是否有硬编码的图片路径（不同语言可能需要不同图片）
5. **文本方向**：是否考虑了 RTL 语言（阿拉伯语、希伯来语）

### 输出格式

```
[h5-forge] 架构设计师：国际化检查

🟡 国际化提醒
1. {文件}:{行号} - 硬编码字符串 "{文本}"，建议提取到 l10n
2. {文件}:{行号} - 日期格式硬编码，建议使用 intl 包
```

### 规则卡字段

在 `team_rules` 下添加：

```yaml
i18n:
  enabled: false                 # 是否启用国际化
  l10n_path: ""                  # 例：src/l10n/
  supported_locales: []          # 例：["zh", "en"]
  confidence: "low|medium|high"
  evidence: []
```

## 无障碍检查

### 检查时机

- 代码生成后：检查 Component 是否有语义标签
- 代码审查时：检查无障碍支持是否完整
- 规则卡有 a11y 要求时：自动启用

### 检查项

1. **语义标签**：交互元素是否有 Semantics 包装
2. **图片描述**：Image 是否有 alt 文本（通过 Semantics）
3. **触摸目标**：可点击元素是否满足最小 48x48 逻辑像素
4. **颜色对比度**：文字和背景是否有足够对比度（WCAG AA 标准 4.5:1）
5. **屏幕阅读器**：重要状态变化是否有语义通知
6. **键盘导航**：是否支持键盘操作（FocusNode、Shortcuts）

### 输出格式

```
[h5-forge] 架构设计师：无障碍检查

🟡 无障碍提醒
1. {文件}:{行号} - 按钮缺少 Semantics 包装
2. {文件}:{行号} - 图片缺少 alt 描述
3. {文件}:{行号} - 触摸目标小于 48x48
```

### 规则卡字段

在 `team_rules` 下添加：

```yaml
accessibility:
  enabled: false                 # 是否启用无障碍检查
  min_touch_target: 48           # 最小触摸目标（逻辑像素）
  contrast_standard: "AA"        # WCAG 对比度标准（AA / AAA）
  screen_reader_support: false   # 是否需要屏幕阅读器支持
  confidence: "low|medium|high"
  evidence: []
```

## 使用原则

- 规则卡中 `i18n.enabled` 或 `accessibility.enabled` 为 false 时，不主动检查
- 用户明确要求时，无论规则卡设置如何都启用检查
- 检查结果只提醒，不自动修改（除非用户明确要求修复）
- 轻量任务不启用检查
