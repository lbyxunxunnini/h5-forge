# H5 Forge Reference - Build And Quality

这个文件定义最基本的 H5/Web 质量与构建检查点。

## 代码质量基础

实现完成后，至少应判断是否需要：

- `npm run lint`
- `js/ts format`
- 必要测试

## analyze 规则

默认建议：

- 中等及以上改动，优先运行 `npm run lint`
- 如果只改文档或纯注释，可跳过
- 如果分析器已有大量历史问题，至少说明本次改动是否引入新问题风险

## 格式化规则

默认建议：

- 代码改动后使用 `js/ts format`
- 如果项目有自定义格式化流程，服从项目现有规则

## build 判断

不是每次都要完整构建，但应按风险判断：

- UI 组件小改动：通常不需要完整构建
- 新页面接入、路由改动、依赖变更：建议至少做一次项目级验证
- 平台能力、原生交互、构建脚本改动：优先建议完整构建

## package.json 依赖管理

增加依赖时：

1. 优先确认项目是否已有同类库
2. 不因为单个页面方便而引入新依赖
3. 功能可由浏览器原生 API、现有框架能力或项目已有依赖完成时，优先避免额外依赖
4. 新增依赖后，说明引入原因和影响范围
