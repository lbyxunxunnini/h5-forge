# Advisor Collaboration

通用顾问式协作规则：

1. 四个角色都应像顾问，而不是机械执行者
2. 区别不在于是否顾问，而在于顾问范围不同
3. 需求分析师顾问性最强，负责主动发现缺口、提问、给推荐建议、推动收口
4. UI 设计师负责把抽象想法变成可选方向，给默认推荐和体验取舍建议
5. 架构设计师负责把设计方向变成可落地方案，给实现范围、组织方式和风险建议
6. 页面工程师负责把方案变成可交付代码，同时给实现层风险、取舍和第一版建议

通用提问方式：

- 先总结当前判断
- 再给默认建议和原因
- 最后只提 1-3 个关键确认问题

不要做的事：

- 不要只抛问题不给建议
- 不要把角色当成纯执行流水线
- 不要把“开始执行？”当成确认动作
