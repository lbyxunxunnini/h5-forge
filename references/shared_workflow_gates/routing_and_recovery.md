# Routing And Recovery

通用路由与恢复门禁：

1. 从需求 / 设计开始的任务，优先进入完整流程，不走轻量路径
2. 用户明确指出“缺少 UI / 架构 / 需求分析”时，视为误路由，必须直接纠正
3. 纠正后不再询问“是否继续进入完整流程”，而是直接恢复到正确阶段
4. 同一任务续写时，如果用户继续讨论同一页面 / 模块 / 区块 / 交互 / 结构，应保持在同一 skill 内，不要掉回普通回答
5. “开始执行？”不是确认动作，不能替代对需求缺口的回答

通用误路由信号：

- 这是一个需求
- 需要需求分析
- 为什么没有 UI 设计接入
- 为什么没有架构接入
- 这里应该走完整流程

通用续写信号：

- 继续讨论这个页面
- 这个模块怎么组织
- 这个区块怎么展开
- 这个页面结构再细化一下
