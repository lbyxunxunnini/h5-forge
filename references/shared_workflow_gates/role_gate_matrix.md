# Role Gate Matrix

## 需求分析师

- 进入：从需求 / PRD / 页面规划开始
- 停机：关键逻辑未补全
- 放行：用户确认状态允许进入下一角色
- 建议预算：`L2 2-4` 问，`L3 4-7` 问，`L4 6-10` 问分阶段执行

## UI 设计师

- 进入：需求已确认，且具备可靠设计输入
- 停机：视觉输入不可读，或多种拆法都合理
- 新项目共创：先过方向确认，再过结构草案确认，才允许展开完整页面结构包
- 放行：结构树、组件边界、交互边界足够支撑架构决策
- 建议预算：`L2 1-2` 次确认，`L3 2-4` 次确认，`L4 3-5` 次确认

## 架构设计师

- 进入：需求和 UI 已足够支持模块归属、文件结构、状态和命名决策
- 停机：高风险公共抽象、不可逆改动、规则冲突
- 放行：规则卡 / 结构方案 / 复用策略明确
- 预算原则：只在存在模块归属、复用、状态管理或命名决策时介入，不为仪式感发言

## 页面工程师

- 进入：已有结构化决策，或属于明确轻量任务
- 停机：实现中发现需求 / UI / 架构缺口或技术风险
- 放行：代码、验证、边界态、规则一致性通过
- 预算原则：轻量任务直做；完整流程中只问阻塞实现的缺口
