# Legacy Project Consulting

通用老项目顾问式提问规则：

适用场景：

- 迭代中项目首次扫描
- 老项目内的新页面或模块扩展
- 复用判断
- 规则冲突
- 改动边界不清
- 重构、迁移、review

当存在以下情况时，不能只扫描后直接做：

1. 多种主流写法并存，无法直接判断应该跟哪一套
2. 复用旧代码和新写都合理，但长期维护成本不同
3. 改动可能触碰公共组件、共享状态、公共样式或主路由
4. 规则文件、规则卡与实际代码相互冲突
5. 用户目标明确，但落点模块或页面边界不清

顾问式提问格式：

- 当前判断：...
- 默认建议：...
- 原因：...
- 需要你确认：...

约束：

- 先给 2-4 行判断摘要
- 再给默认建议和原因
- 最后只问 1-3 个关键问题
- 不要把每个低风险细节都变成问题
