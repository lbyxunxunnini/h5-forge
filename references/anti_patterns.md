# H5 Forge Reference - Anti Patterns

以下是 H5 Forge 默认需要警惕的常见 H5/Web 反模式。

## error

- 在渲染过程中直接触发状态更新，造成重复渲染或循环渲染
- 页面层直接发起数据层请求且绕过既定架构
- 为了复用而抽出高度参数化且语义空泛的公共组件

## warning

- 异步请求返回后未判断组件是否仍然有效
- 初始化阶段读取尚未稳定的路由、store 或 DOM 状态
- 未处理 Promise 或异步错误分支
- 长列表不分页、不懒加载、不虚拟化，直接一次性渲染所有节点
- 把跨页面共享业务状态硬塞进局部页面 state

## info

- 用户可见字符串未国际化
- 稳定子组件没有拆分，导致父级状态变化时大范围重渲染
- 明显可提取的重复 UI 片段仍然散落在页面中
