# H5 Forge Cheatsheet

## 常见任务怎么问

| 任务 | 推荐说法 |
|------|----------|
| 小改动 | `h5f-fast 改一下登录页按钮文案和颜色` |
| 新页面 | `h5f- 新增订单列表页，接入现有路由和接口层` |
| 全自动页面 | `h5f-a 做会员中心页，缺失信息按项目主流方案推进` |
| 扫描老项目 | `h5f- 先扫描这个 H5 项目，生成规则卡草案，不写代码` |
| 新项目共创 | `h5f- 我有个新 H5 项目想法，先共创需求和页面结构` |
| 代码审查 | `h5f- review 这个模块的状态管理和组件边界` |
| 性能优化 | `h5f- 优化首页首屏加载和列表渲染性能` |
| 迁移 | `h5f- 把这个模块从 Context 迁移到 Zustand，先给风险判断` |

## 入口选择

- `h5f-`：默认选择，稳妥。
- `h5f-fast`：你明确想快，任务较小或中等。
- `h5f-a`：你接受 AI 采用推荐方案自动补齐缺口。

## 你会看到什么

```text
[h5-forge] 模式：中等任务 / 快速
- 扫描摘要：React + Zustand + React Router + Axios
[h5-forge] 页面工程师：完成最小改动，验证通过
```

```text
[h5-forge] 模式：页面开发 / 全自动
- auto_assumption：未提供空状态样式，沿用现有 EmptyState 组件
[h5-forge] 验证工程师：最小验证通过
```
